TASK 1: PDF REPORT EXPORT FOR DASHBOARD
================================================================================

CONTEXT:
FYPAuto (GeniPilot) is an AI-powered browser automation platform built with
FastAPI (main.py on port 8000), an Agent Server (agent_server.py on port 5005),
MongoDB (async motor), and a Chrome Extension (Manifest V3 side panel).

The dashboard (templates/dashboard.html) already has:
- Automation history table with search, filter, status badges, rerun, delete
- Scheduled tasks section with CRUD operations
- Stats cards: total tasks, success rate, scheduled count, next run
- Usage monitoring: daily task usage vs limit, scheduled tasks vs limit
- Charts: line chart (task timeline via Chart.js), pie chart (status breakdown)
- Profile section, notification settings, plan & billing section

The backend (main.py) already has these relevant endpoints:
- GET /api/automation-history — returns user's automation history from MongoDB
- GET /api/scheduled-tasks — returns user's scheduled tasks
- GET /api/user-stats — returns aggregated stats (total, success rate, etc.)
- GET /api/export-history — exports history as CSV/JSON (excludes thumbnails)

CRITICAL RULE: Do NOT break or alter any existing functionality — the dashboard
UI, WebSocket connections, authentication, automation execution, HITL modals,
scheduled tasks, plan limits, history table, charts, profile management,
notification settings, extension chat, voice assistant, image module, or any
other feature must all continue to work exactly as they do now.

================================================================================
FEATURE DESCRIPTION
================================================================================

Add a "Download PDF Report" button to the dashboard that generates a
professional PDF report of the user's automation activity. This report should
be useful for project evaluators/professors to see a summary of the system's
capabilities and the user's usage.

================================================================================
PDF REPORT CONTENT
================================================================================

The PDF report should include the following sections:

1. HEADER
   - Report title: "GeniPilot — Automation Report"
   - Generated date and time
   - User name and email

2. SUMMARY STATISTICS
   - Total automations run
   - Success rate (percentage)
   - Total scheduled tasks
   - Average task duration
   - Current plan name
   - Date range of the report

3. TASK STATUS BREAKDOWN (Table)
   - Number of completed tasks
   - Number of failed tasks
   - Number of cancelled tasks

4. RECENT AUTOMATION HISTORY (Table — last 20 tasks)
   - Task name
   - Status (completed/failed/cancelled)
   - Date & time
   - Duration
   - Brief result summary (truncated to 100 chars)

5. SCHEDULED TASKS (Table — all active)
   - Task name
   - Frequency (once/hourly/daily/weekly/monthly)
   - Next scheduled run
   - Last run status

6. FOOTER
   - "Generated by GeniPilot — AI-Powered Browser Automation"
   - Page numbers

================================================================================
IMPLEMENTATION PLAN
================================================================================

APPROACH: Use a lightweight Python PDF library. Choose ONE of:
  - reportlab (most popular, no extra system deps)
  - fpdf2 (simple, lightweight)
  - weasyprint (HTML to PDF, but requires system deps — AVOID)

Recommended: Use "reportlab" or "fpdf2" — add it to requirements.txt and
pyproject.toml dependencies.

--- BACKEND (main.py) ---

1. Add new endpoint: GET /api/export-pdf-report
   - Requires authentication (same session check as other dashboard endpoints)
   - Fetches user stats, automation history (last 20), and scheduled tasks
   - Generates PDF in memory using BytesIO
   - Returns PDF as StreamingResponse with:
     - Content-Type: application/pdf
     - Content-Disposition: attachment; filename="genipilot-report-{date}.pdf"

2. PDF generation function: generate_pdf_report(user_data, stats, history, scheduled_tasks)
   - Takes the fetched data and builds the PDF
   - Use clean formatting: tables with alternating row colors, headers, etc.
   - Keep it professional and clean — no emojis in the PDF
   - Handle edge cases: no history yet, no scheduled tasks, etc.

--- FRONTEND (templates/dashboard.html) ---

3. Add a "Download Report" button in the dashboard
   - Place it near the existing export buttons (CSV/JSON) in the history section
     OR in the stats/overview section at the top
   - Style it consistently with existing buttons (use existing CSS classes)
   - On click: window.location.href = '/api/export-pdf-report'
     (this triggers the download directly)
   - Add a small loading indicator while PDF generates (optional)
