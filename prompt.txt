================================================================================
CHROME EXTENSION — "SUMMARIZE THIS PAGE" SENDS WRONG TAB / BLANK TAB BUG
================================================================================

CONTEXT:
GeniPilot is a Chrome extension (Manifest V3 side panel) that lets users type
automation commands like "summarize this page". The extension sidebar
(sidebar.js) communicates with the agent server (agent_server.py) via WebSocket.
The agent server runs browser automation using browser-use connecting to Chrome
via CDP (Chrome DevTools Protocol) at localhost:9222.

CRITICAL RULE: Do NOT break or alter any existing functionality — message
sending, WebSocket streaming, image attach, TTS, templates, stop/cancel,
plan limits, scheduled tasks, re-run, HITL input modals, onboarding tour,
or any other feature must all continue to work exactly as they do now.

================================================================================
BUG DESCRIPTION
================================================================================

When the user has multiple Chrome tabs open (e.g., Tab 1: YouTube, Tab 2:
Gmail, Tab 3: SalmanFoods, Tab 4: Wikipedia) and is CURRENTLY VIEWING the
SalmanFoods tab, then types "summarize this page" in the extension sidebar:

  BUG 1: Agent opens a NEW BLANK tab (about:blank) and returns empty content
  BUG 2: Agent summarizes the WRONG tab (e.g., YouTube instead of SalmanFoods)
  EXPECTED: Agent should summarize the SalmanFoods tab — the one the user
            is currently looking at

This affects ALL "current page" commands — not just summarize:
  - "summarize this page"
  - "extract all links from this page"
  - "what is this page about"
  - "fill out the form on this page"
  - Any task referencing "this page", "current page", "this tab", "this site"

================================================================================
ROOT CAUSE ANALYSIS
================================================================================

There are TWO separate failures causing this bug:

─── FAILURE 1: Extension never sends current tab URL ───────────────────────────

File: extension/sidebar.js
Function: sendMessage() (inside initChat())
Lines: ~1118-1128

The WebSocket payload sent to the server is:
    const payload = { message: message };
    if (imageData) { payload.image = imageData; }
    websocket.send(JSON.stringify(payload));

The payload ONLY contains { message: "summarize this page" }. It does NOT
include the URL or title of the tab the user is currently viewing.

The extension has access to Chrome APIs (chrome.tabs.query) that can retrieve
the active tab's URL and title, but this data is NEVER fetched or sent.

─── FAILURE 2: Tab selection falls through to "create new tab" ─────────────────

File: agent_server.py
Function: select_automation_tab() → _find_relevant_tab()
Lines: ~766-864

The tab selection logic works by extracting DOMAIN KEYWORDS from the task text:
    task = "summarize this page"
    → No explicit URL found (no https?:// match)
    → No keyword match in _DOMAIN_KEYWORDS (no "youtube", "gmail", etc.)
    → target_domains = empty set
    → _find_relevant_tab() returns None (line 788)

Since no relevant tab is found, select_automation_tab() hits Rule 3:
    "No relevant tab found — creating a new tab."
    → Creates about:blank tab
    → Agent attaches to blank tab → gets nothing to summarize

Even if the tab selection somehow picks a tab, it picks based on domain
keywords in the task text, NOT based on which tab the user is actually viewing.

================================================================================
THE FIX — TWO PARTS
================================================================================

─── PART 1: Extension sends current tab URL + title with every message ─────────

File to modify: extension/sidebar.js
Function to modify: sendMessage() inside initChat()

BEFORE building the WebSocket payload, query the active Chrome tab:

    // Get the active tab URL and title
    let currentTabUrl = '';
    let currentTabTitle = '';
    try {
        const [tab] = await chrome.tabs.query({
            active: true,
            currentWindow: true        // ← IMPORTANT: same window the user is in
        });
        if (tab && tab.url) {
            currentTabUrl = tab.url;
            currentTabTitle = tab.title || '';
        }
    } catch (e) {
        console.log('Could not get active tab:', e);
    }

Then include it in the payload:

    const payload = {
        message: message,
        current_tab_url: currentTabUrl,
        current_tab_title: currentTabTitle
    };
    if (imageData) { payload.image = imageData; }

IMPORTANT: The sendMessage function must become `async function sendMessage()`
(it already is async in the current code, so this is fine).

IMPORTANT: The extension manifest.json must include the "activeTab" permission
in the "permissions" array. Check if it's there; if not, add it:

    "permissions": ["sidePanel", "storage", "identity", "activeTab"]

ALSO: host_permissions already includes "http://localhost:8000/*" and
"http://localhost:5005/*", but for querying ANY tab URL, the extension needs
broad host permissions or the "tabs" permission. Add "tabs" to permissions:

    "permissions": ["sidePanel", "storage", "identity", "activeTab", "tabs"]

─── PART 2: Server uses current_tab_url for tab selection ──────────────────────

File to modify: agent_server.py

2A. Extract current_tab_url from the WebSocket message
    In the WebSocket handler (websocket_chat function), after parsing
    message_data, extract:

        current_tab_url = message_data.get("current_tab_url", "")
        current_tab_title = message_data.get("current_tab_title", "")

    Pass current_tab_url to execute_automation_task() and then to
    select_automation_tab().

2B. Modify select_automation_tab() signature and logic
    Change: async def select_automation_tab(task: str) -> str:
    To:     async def select_automation_tab(task: str, current_tab_url: str = "") -> str:

    Add a NEW Rule 0 — HIGHEST PRIORITY — before Rule 1 (new tab phrases):

        # Rule 0 — If current_tab_url is provided AND the task references
        # "this page"/"current page"/"this tab"/"this site", find and activate
        # the tab matching that URL.
        _CURRENT_PAGE_PHRASES = [
            "this page", "current page", "this tab", "this site",
            "this website", "the page", "the current",
        ]

        if current_tab_url and any(p in task.lower() for p in _CURRENT_PAGE_PHRASES):
            valid_tabs = await _get_valid_browser_tabs()
            for tab in valid_tabs:
                if tab.get("url", "") == current_tab_url:
                    print(f"📋 [TabSelect] Matched current tab by URL: {tab.get('url', '')[:70]}")
                    await _activate_existing_tab(tab["id"])
                    return tab["id"]
            # Fallback: URL not found in CDP targets (unlikely) — continue
            # to other rules

    This ensures that when the user says "summarize this page" and the
    extension sends current_tab_url="https://salmanfoods.com/...", the server
    finds exactly that tab and activates it.

2C. Modify execute_automation_task() to accept and forward current_tab_url
    Change the function signature to include current_tab_url parameter.
    Pass it through to select_automation_tab().

2D. Modify the Automation tool function to accept and forward current_tab_url
    The @function_tool Automation() callable needs to receive current_tab_url
    from the calling context. Since tool functions are called by the OpenAI
    Agent SDK, you cannot easily add parameters. Instead, store current_tab_url
    in a global/context variable (similar to _current_automation_user):

        _current_tab_url = ""

    Set it before calling Runner.run_streamed() in the WebSocket handler.
    Read it inside execute_automation_task().

2E. Enrich the task prompt with the actual URL
    In execute_automation_task(), after the select_automation_tab() call,
    if current_tab_url is provided and the task mentions "this page", prepend
    context to the processed_task:

        if current_tab_url and any(p in processed_task.lower() for p in _CURRENT_PAGE_PHRASES):
            processed_task = (
                f"[CONTEXT: The user is currently viewing: {current_tab_url} "
                f"(title: {current_tab_title}). "
                f"Perform the following task on THIS specific page.]\n\n"
                f"{processed_task}"
            )

    This gives the browser-use agent explicit URL context so it navigates to
    the correct page even if tab selection had issues.
